Two Authentication Scenarios

Scenario A: External App ‚Üí Databricks

Your application runs OUTSIDE Databricks and needs to call Databricks APIs.

  • OAuth Token Exchange (Token Federation)
  • Native OAuth with Entra (Azure)
  • IdP tokens exchanged for Databricks tokens
  • Server-side token handling

Scenario B: Agent ‚Üí Resources

Your agent runs ON Databricks (Model Serving) and needs to access resources.

  • Automatic Authentication Passthrough
  • On-Behalf-Of-User (OBO)
  • Manual Authentication
  • Declared at logging time

üóÉÔ∏è Lakebase: Agent Memory & OLTP State

Lakebase is a fully-managed Postgres OLTP database for agent memory (short-term & long-term), session state, and fast governed data access. Supports both OAuth and native Postgres authentication.

OAuth (U2M / M2M)
  • 1-hour tokens as DB password
  • User, Group, or Service Principal
  • Requires SSL connection
Native Postgres Password
  • For apps that can't rotate tokens
  • Standard CREATE ROLE
  • Enable via instance settings
