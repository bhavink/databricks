What are Databricks Apps?

Databricks Apps is a managed platform for hosting web applications directly within your Databricks workspace.

Build data applications, AI demos, dashboards, and internal tools ‚Äî all running on serverless compute with built-in security.

Key benefit: No separate infrastructure to manage. Your app inherits Databricks security, authentication, and governance automatically.

Supported Frameworks

Databricks Apps supports both Python and Node.js frameworks:

  • Streamlit ‚Äî Data apps & dashboards
  • Dash ‚Äî Interactive visualizations
  • Gradio ‚Äî ML/AI demos & chat interfaces
  • React/Angular/Svelte ‚Äî Custom web apps
Deployment: Push your code ‚Üí Databricks builds and serves it on serverless infrastructure.

Native OAuth Authentication

Apps running on Databricks get automatic OAuth ‚Äî no manual token handling required.

When a user accesses your app, they authenticate through Databricks. The platform provides your app with tokens to access workspace resources on behalf of that user.

Zero configuration: OAuth is handled by the platform. Just use the Databricks SDK in your app code.

Direct Access to Resources

Your Databricks App can directly access all workspace resources:

  • SQL Warehouses ‚Äî Run queries on lakehouse data
  • Model Serving ‚Äî Call deployed ML models & agents
  • Unity Catalog ‚Äî Browse and query governed tables
  • Jobs & Workflows ‚Äî Trigger and monitor pipelines
Resources are accessed with the user's identity ‚Äî UC permissions are enforced automatically.

Unity Catalog Integration

Every data access from your app flows through Unity Catalog governance:

  • Permissions ‚Äî Users only see data they're authorized to access
  • Row filters ‚Äî Automatic row-level security
  • Column masks ‚Äî Sensitive data is masked per policy
  • Audit logs ‚Äî All access is tracked
Build once, govern always: Your app code stays simple while UC handles fine-grained access control.

User Identity Propagation

When a user interacts with your app, their identity flows through to all backend resources:

User ‚Üí App ‚Üí SQL Warehouse ‚Üí UC ‚Üí Data

This ensures that data access respects each user's permissions ‚Äî two users using the same app may see different data based on their roles.

Example: A finance user sees all revenue data; a regional manager sees only their region's data ‚Äî from the same dashboard.

vs. External App Hosting

Compare Databricks Apps with hosting your app externally:

  • No token exchange needed ‚Äî OAuth is automatic vs. manual federation
  • No infrastructure ‚Äî No VMs, containers, or cloud setup
  • Built-in security ‚Äî Network isolation, secrets management included
  • Direct UC access ‚Äî No VPN or firewall configuration
Trade-off: External hosting offers more flexibility; Databricks Apps offers simpler security and governance.
